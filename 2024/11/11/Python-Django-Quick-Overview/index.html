Python & Django Quick Overview

William Fan
2024-11-11

Python Package is a container for a bunch of modules

  • a python Module is a signle source file
  • using package to reduce naming pollution and improve code isolation
  • eg. __init__.py tells python its containing folders should be treated like a package

In Django, we use two different commands to organize our projects.

  1. django-admin startproject todo_project ., we need this command to create a project folder that provides project-wide information, which means general project settings, information that a web server will need will be place at here
    1. setting.py is a project-wide configurations (applications/databases)
    2. urls.py has a list of URLs that the server must listen for (Django’s URL dispatcher uses the elements in urlpatterns to decide how to handle incoming requests)
  2. django-admin startapp todo_app, we need this command to create an app folder where the app will actually be run
    1. migrations/, the subfolder holds the database migration information
    2. models.py, we define the data model for the app (like Rails model)
    3. views.py, we design the control logic for the app (like Rails controller)
    4. templates/, files contain HTML markup

In Django, we use these two commands to execute database migrations:

  1. python manage.py makemigrations todo_app, with makemigrations, you are telling Django, you’ve changed the model, and you’d like to record this change.
  2. python manage.py migrate, with migrate, you put those changes into effect.

How does this request and response cycle works?

  1. The server receives a GET request from a broswer, it creates an HTTPrequest object and using the URL dispatcher to find ListListView
  2. The ListListView (a ListView based on TodoList model) fetches all the TodoList records from database, turns them into Python objects and append them to object_list (default)
  3. The ListListView pass object_list to the template engine, which use the specified template, index.html
  4. The template engine builds the HTML code by using the passed-in data to populate HTML elements and combining it with base.html
  5. The view constructs an HTTPresponse object, containing the HTML and returns to server
  6. The server turns the HTTPresponse into an HTTP message and sends it back to the broswer
  7. The broswer fully render the response to the user.
